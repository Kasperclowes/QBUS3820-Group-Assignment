Feature engineering: 
Most important predictors to focus on for our problem: 

1. Transactions (critical)
2. Customer level aggregates 
3. Demographics 
4. Coupons and coupon redemption 
Moderate/less important predictors: 
5. Promotions
6. Campaing and campaign Descriptions
7. Individual products 

1. Transactions (CRITICAL): For each houshold id: 

 - Recency → weeks/days since last purchase: 
        1. In transactions df: For each houshold id, 
        find the latest and second latest timestamp in date on column transaction_timestamp
        2. Find the difference in weeks/days between the transactions
        3. Output as a numerical value in a new added column called recency 

- Frequency → number of purchases in last X weeks/months 

        1. In transactions df: For each houshold id 
        find the number of transactions in the last X weeks/months (this is a larger timeperiod than recency)
        2. Look for a specific chosen timestamp, any transactions commited after this timestamp is accounted for in frequency parameter
        3. Output as a numerical value in a new added column called frequency 

- Monetary value → spend trends

In transactions df:
        1.  for each household id compute total spend (sum of all purchases), 
        Average order value (AOV) (total spend/number of transactions) and weekly/monthly spend week/month to get an idea on 
        how much the specific household id is usually spending, a high or low spending household. 
        2. Option A Spend trends: For each household id compute spend trends. Define a recent to past spend behaviour (ex total spend previous 4 weeks vs past 5-8 weeks)
        Spend Trend= (Recent Spend - Past spend)/Past Spend. 
        Negative → spending is dropping (churn risk)
        Positive → increasing engagement
        3. Option B Moving average trend: (for example per 4 weeks average) 
        how does the average transactions per houshold over 4 weeks develop over time
        Increasing average or decreasing? Could tell us if a houshold is about to churn 
        
        Spend alone can mislead:

Someone might spend less just because they visit less: 

↓ frequency + ↓ spend → strong churn signal
↓ frequency + same AOV → weaker churn signal

- visit decline rate: 

Visit Rate= (Recent visits-past visits)/ past visits 
	​
        Negative visit rate- High decline → strong churn risk
        Near 0 → stable
        Positive visit rate → growing engagement




2. Customer-Level Aggregates (Derived from Transactions)
Rolling purchase frequency (e.g., last 4 weeks vs previous 4 weeks)
Visit decline rate
Spend decline rate

3. Demographics (Moderate–High Value)
Useful for segmentation and interaction effects, not as standalone predictors.
Income vs churn rate
Household size vs churn
Age groups vs churn

4. Coupons & Coupon Redemption (High Value)
Very relevant for retention modelling.
Redemption rate per customer
Time since last redemption
Do redeemers churn less?
Response to past offers

5. Promotions- Only useful if tied to behaviour.
Did purchases occur during promotions?
Does reliance on promotions correlate with churn?
Alone → weak
Combined with transactions → useful

6. Campaigns & Campaign Descriptions
Engagement history
Marketing responsiveness
EDA:
Who received campaigns vs who returned?
Campaign exposure vs inactivity

7. Products- Only useful after aggregation.
Raw product-level EDA is usually not helpful for churn directly.

Category diversity (variety of products bought)
Dependency on certain categories
Drop in key category purchases
